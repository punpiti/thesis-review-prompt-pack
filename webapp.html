รายการ prompt ที่ควรมี

ไม่ต้องใช้ทุก prompt ในครั้งเดียว ให้เลือกตามปัญหาที่ต้องตรวจ แล้วแทนช่อง [วาง...] ด้วยข้อความจากตำแหน่งที่ระบุ

Prompt 1: ตรวจว่าเอกสารขาดอะไรและจำเป็นแค่ไหน

ฉันต้องการให้ช่วยตรวจร่างวิทยานิพนธ์/manuscript แบบ reviewer

งานแรก:
- อ่านข้อความจากเอกสารที่ฉันวางให้
- บอกว่าข้อมูลอะไรขาด
- แยกว่าข้อมูลที่ขาดนั้น จำเป็นมาก / ควรมี / ไม่จำเป็นในรอบนี้
- ถ้าข้อมูลอยู่ในเอกสารแล้ว ให้สรุปจากเอกสาร ไม่ต้องถามให้ฉันกรอกซ้ำ
- ถ้าข้อมูลไม่พอและจำเป็นต่อการตรวจจริง ให้ถามกลับเฉพาะสิ่งนั้น

กฎ:
- ห้ามแต่งข้อมูลเกินหลักฐานที่ให้ไว้
- ห้ามแต่ง citation, DOI, venue ranking, metric, result, IRB/legal requirement หรือข้อกฎหมายเอง
- ถ้าข้อมูลไม่พอ ให้ถามกลับ
- ขอ feedback ภาษาไทย แต่คง technical terms ภาษาอังกฤษได้
- ทุกประเด็นสำคัญต้องบอกว่าดูจากข้อมูลส่วนไหน เช่น section, paragraph, table หรือ figure
- แยก output กับ outcome และตรวจว่างานไล่รอยจาก problem -> prior attempts/evidence -> unresolved need -> objective/RQ -> method -> evidence -> output -> outcome ได้หรือไม่

[วางข้อความจากเอกสารเท่าที่ส่งได้ในรอบแรก:
- Title page
- Abstract
- Introduction หรือ Problem statement
- Objectives/RQs
- Table of contents ถ้ามี
- Methodology summary ถ้ามี]

Prompt 2: ตรวจ title, abstract, problem, objectives

ช่วยตรวจเฉพาะ title, abstract, problem statement และ objectives/RQs ต่อไปนี้

ขอ output เป็น:
1. title กับ abstract สอดคล้องกันหรือไม่
2. problem มีน้ำหนักพอหรือไม่
3. objectives/RQs มาจาก problem หรือไม่
4. มี claim ใดแรงเกินหลักฐานหรือไม่
5. top 5 fixes ก่อนส่งอาจารย์

ห้าม rewrite ทั้งหมด ให้เสนอเฉพาะจุดที่ควรแก้และเหตุผล

[วางจาก manuscript/thesis:
- Title page: title
- Abstract section: abstract ทั้งหมด
- Introduction / Problem statement: ย่อหน้าที่อธิบายปัญหาและ research gap
- Objectives / Research questions section: objective หรือ RQ ทุกข้อ]

Prompt 3: ตรวจ literature review แบบประหยัด quota

ส่งเฉพาะหัวข้อย่อยของ related work, 1-2 ย่อหน้าสำคัญต่อหัวข้อ และ reference list/BibTeX เฉพาะรายการหลัก

ช่วยตรวจ literature review ส่วนนี้แบบ reviewer

ขอให้เน้น:
1. literature review เป็น synthesis หรือแค่ summary ทีละ paper
2. มี claim ที่ควร cite แต่ยังไม่มี citation หรือไม่
3. citation ใดดูไม่ตรง claim
4. reference ควร cluster เป็นกลุ่มอะไรบ้าง
5. ควรมี comparison matrix แบบใด
6. มี direct prior work, cross-domain support, foundational support, target-venue anchor แยกชัดหรือไม่

ตอบแบบสั้นเป็นตารางและ bullet

[วางจาก manuscript/thesis:
- Related Work / Literature Review section: หัวข้อย่อยทั้งหมด และย่อหน้าสำคัญ 1-2 ย่อหน้าต่อหัวข้อ
- Reference list หรือ BibTeX เฉพาะรายการที่ถูก cite ใน section นี้
- ถ้ามี comparison table เดิม ให้วางด้วย]

Prompt 4: ตรวจ method, data, baseline, results

ช่วยตรวจ methodology และ results ต่อไปนี้

ขอให้ตอบ:
1. method แก้ problem/objective จริงหรือไม่
2. baseline พอหรือไม่ ถ้าไม่พอควรมีอะไร
3. metrics เหมาะสมหรือไม่
4. results ตอบ objectives/RQs หรือไม่
5. มี result claim ใดที่ยังไม่มี raw result/script/error analysis รองรับ
6. top 5 blocking issues

ห้ามแต่งผลทดลองเพิ่ม

[วางจาก manuscript/thesis:
- Methodology / Proposed Method section
- Dataset / Data Collection section
- Baseline / Comparative Methods section
- Evaluation Metrics section
- Results section เฉพาะตาราง/รูป/ย่อหน้าที่รายงานผล
- ถ้ามี appendix หรือ supplementary result ที่เกี่ยวข้อง ให้วางสรุปสั้น ๆ]

Prompt 5: ตรวจ ethics / IRB / legal compliance

ช่วย flag ประเด็น ethics/IRB/legal compliance จากข้อมูลนี้

กฎ:
- ห้ามให้ legal advice
- ห้ามสรุปแทนว่าไม่ต้องขอ IRB ถ้าไม่มีหลักฐานจากสถาบัน
- ให้บอกว่าควรถามอาจารย์/IRB office/venue/institution เรื่องใด

ข้อมูล:
[วางจาก manuscript/thesis หรือเอกสารประกอบ:
- Data Collection / Participants / User Study section
- Consent / anonymization / de-identification description
- Dataset source, license, Terms of Service หรือ scraping/platform details
- Ethics statement / IRB approval / exemption ถ้ามี]

ขอ output:
1. ประเด็นที่อาจต้องมี IRB/ethics approval หรือ exemption
2. consent/anonymization/de-identification ที่ควรมี
3. license/copyright/Terms of Service/data ownership ที่ต้องตรวจ
4. ข้อความที่ manuscript ควรมี เช่น ethics statement, data availability, consent statement

Prompt 6: ตรวจภาษาและ document hygiene

ช่วยตรวจ document hygiene ของข้อความนี้

เน้น:
1. สะกดผิดหรือ typo มากเกินเหตุ
2. ใช้ชื่อย่อโดยไม่ define
3. ปนภาษาไทย/อังกฤษเกินจำเป็น
4. อ้างถึงสิ่งที่อยู่ด้านหลังที่ผู้อ่านยังไม่เห็น
5. มีร่องรอยใช้ AI แบบ generic, claim ลอย, คำแปลแปลก หรือ terminology ไม่สม่ำเสมอ

ห้าม rewrite ทั้งหมด ให้ list ปัญหาและตัวอย่างเท่านั้น

[วาง section ที่ต้องการตรวจ โดยระบุชื่อ section ก่อนข้อความ เช่น "Section 1 Introduction" หรือ "Section 3 Methodology"]

Prompt 7: ถ้า quota เหลือน้อยมาก

ตรวจเฉพาะ blocking issues 5 ข้อแรกจากข้อความนี้
ต้องอ้างตำแหน่งจากหัวข้อ/ย่อหน้า
ห้าม rewrite ทั้งบท
ห้ามแต่งข้อมูลเกินหลักฐานที่ให้ไว้
ห้ามแต่ง citation, DOI, result, venue ranking, หรือ IRB/legal requirement เอง

[วาง section ที่กังวลที่สุด 1 section เท่านั้น พร้อมชื่อ section และเลขหน้า/หัวข้อถ้ามี]
สำหรับสนับสนุนการตรวจงานวิชาการ ไม่ใช่ตัวแทนของอาจารย์ที่ปรึกษา reviewer หรือการพิจารณา IRB/legal compliance จริง
